# ready-compact — compaction 전 상태 저장 + resume 프롬프트

컨텍스트가 곧 요약(compaction)될 때, 다음 세션이 **휘발성 맥락 없이도** 작업을 이어가도록
준비한다. compaction은 대화를 요약하지만 "지금 어디까지 했고 다음 한 수가 무엇인지"가 흐려질
수 있으므로, 그 핵심을 (1) 내구 메모리에 저장하고 (2) 붙여넣기용 resume 프롬프트로 출력한다.

## 언제 쓰나
- 컨텍스트가 한계에 가까워 곧 compaction 될 때(또는 사용자가 미리 준비를 요청할 때).
- 긴 멀티스텝 작업을 중간에 끊었다가 이어야 할 때.
- 사용자가 "ready-compact", "compact 준비", "이어서 할 수 있게 준비" 등으로 요청할 때.

## 쓰지 않을 때
- 작업이 이미 끝났고 이어갈 게 없을 때(그냥 마무리 보고로 충분).
- 코드/깃/CLAUDE.md/메모리가 그대로 담는 사실의 단순 복제(중복).

## 워크플로우

### 1) 상태 스냅샷 수집 (휘발성 맥락 위주)
지금 대화에서 아래를 추린다 — **메모리/플랜에 이미 있는 건 링크만**, 없는 휘발성만 본문에:
- **목표(Goal)**: 사용자의 원 요청 한 줄.
- **현재 단계(Phase)**: 전체 흐름 중 지금 위치.
- **완료(Done)**: 커밋 해시 + 핵심 산출물(파일/마이그레이션/모니터). 가능하면 `git log --oneline`로 확정.
- **현재 상태(State)**: 서버/DB/테스트의 *검증된* 상태(예: "37 테스트 PASS, 서버 재기동 정상, 모니터 8/8 PASS").
- **다음 한 수(Next)**: compaction 직후 실행할 **구체적** 액션 1~3개(명령/파일/대상 포함). 사용자 결정 대기면 그 사실을 명시.
- **미결 결정(Open)**: 사용자 판단이 필요한 분기(있으면).
- **함정·제약(Gotchas)**: 이 작업에서만 비자명한 것(세션 한정 명령, 순서 의존, 미커밋 여부, 운영 미반영 등).
- **검증 명령(Verify)**: 이어받은 사람이 현 상태를 즉시 재확인할 복붙 명령.

### 2) 내구 저장
프로젝트 메모리 디렉터리(`<memory>/`)에 resume 노트를 기록한다(메모리는 compaction을 넘어
세션마다 로드됨). **기존 메모리/플랜과 중복은 피하고**, 관련 메모리는 `[[slug]]`로 링크.
- 파일: `<memory>/resume-<topic-slug>.md` (frontmatter: `type: project`).
- 본문: 위 스냅샷(특히 Next/Open/Gotchas/Verify)을 응축. 영속 상세는 링크.
- `MEMORY.md`에 한 줄 포인터 추가/갱신.
- 작업이 git 미커밋 상태면 그 사실과 파일 목록을 명시(다음 세션이 덮어쓰지 않도록).

### 3) resume 프롬프트 생성
compaction 이후 **그대로 붙여넣으면** 작업이 이어지는 자급식 프롬프트를 코드블록으로 출력.
템플릿:

```
[RESUME] <목표 한 줄>. 진행 메모리: <memory>/resume-<slug>.md (+ MEMORY.md, [[관련 slug]]).
현재까지: <완료 요약 + 커밋 해시>. 상태: <검증된 상태>.
다음 할 일: <구체 액션 1~3, 또는 "사용자 결정 대기: <옵션들>">.
미결: <사용자 판단 분기 / 없으면 생략>.
먼저 현 상태 확인:
  <verify 명령 복붙>
제약: <환경/순서/미커밋/운영미반영 등 핵심 함정>. (상세는 위 메모리 파일 참조.)
```

### 4) 보고 (한글)
- 저장한 resume 파일 경로 + MEMORY.md 갱신 여부.
- resume 프롬프트(코드블록)를 마지막에 제시.
- "compaction 후 이 프롬프트를 붙여넣으면 이어집니다" 안내.

## 작성 원칙
- **휘발성 우선**: 메모리/플랜/깃이 담는 영속 사실은 링크, compaction이 잃을 '다음 한 수'를 본문에.
- **자급식 resume**: 프롬프트만 보고도 검증→다음 액션이 가능해야 함(메모리 파일은 백업/상세).
- **증거 기반**: 커밋 해시·테스트 수·모니터 결과 등 *검증된* 값으로. 추측 금지.
- **덮어쓰기 방지**: 미커밋 변경/운영 미반영이 있으면 반드시 명시.

ARGUMENTS: $ARGUMENTS (선택: 이어갈 작업의 토픽/슬러그 힌트. 없으면 현재 대화에서 추론.)
