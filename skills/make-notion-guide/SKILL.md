# Make-Notion-Guide: 노션 양식 개발자 가이드 작성

프로젝트의 **실제** API/인터페이스를 노션(체크박스 토글) 양식의 개발자 호출 가이드 문서로 만든다.
GitHub 에서도 깨지지 않게 렌더되며, 예시는 라이브로 검증된 값만 싣는다.

## 원칙 (어기면 "잘못된 문서"가 나온다)

1. **실제 인터페이스를 문서화하라 — 템플릿/남의 문서 내용 복붙 금지.**
   다른 시스템의 host·endpoint·auth·응답을 그대로 두고 이름만 바꾸면 "이름만 갈아끼운 남의 문서"가 된다.
   반드시 대상 프로젝트의 코드·spec 에서 호출 규약을 직접 확인한다.
2. **예시 동작은 라이브로 검증하라 — action/출력 단정 금지.**
   각 예시를 실제로 호출해 결과(action·마스킹·차단)를 확인한 뒤 라벨을 단다.
   예시 값도 실제 형식 길이·패턴을 충족해야 한다(짧은 더미값은 탐지기에 안 걸려 오해를 부른다).
3. **호출 예시 JSON 은 실제 응답을 trim 해서 싣고, 생략한 필드는 명시한다.**
4. **시크릿은 플레이스홀더·환경변수.** 토큰·키 하드코딩 금지(`$APP_ID`, `<발급받은 키>`).

## 양식 (노션 체크박스 토글)

- 최상위 섹션은 `- [ ]` 토글, 하위 내용은 2칸 들여쓰기(코드펜스 포함)
- **H1 제목 없음** — 노션 페이지 제목은 본문 밖이므로 `- [ ] **개요**` 로 바로 시작
- 코드펜스 언어는 내용에 맞게: `bash`(셸/curl) · `json`(응답) · `text`(URL 등). python/jsx 금지(GitHub 렌더 깨짐)
- 표준 섹션 순서: **개요 → Repo → 접근정보 → (테스트 UI) → 예시 질문 → 호출 예시**

## 절차

1. **인터페이스 발굴**: README·`spec/`·라우트 코드에서 host/port·인증 방식·엔드포인트·요청/응답 스키마·오류코드 확인.
2. **섹션 초안**: 위 표준 섹션으로 뼈대 작성.
3. **예시 라이브 검증**: 카테고리별 입력을 실제 호출 → action/출력 확인 → 표에 실측 라벨.
4. **호출 예시**: 실제 curl + 실제 응답(trim). 끝에 "(…일부 필드 생략)" 표기.
5. **조립**: 양식 규칙대로.
6. **검증**(아래 체크리스트).
7. **저장·링크**: `docs/<name>.md` 로 저장, README·설계노트에서 링크(링크 문구도 내용과 일치시킬 것).

## 스켈레톤 (대상 프로젝트 값으로 채운다)

````markdown
- [ ] **개요**

  <한 줄 정의>

- [ ] **Repo**

  <repo URL>

- [ ] **접근정보**

  ```bash
  - 외부 노출: <host:port>
  - 인증: <방식 — 예: 헤더 APP-ID / Bearer>     # 시크릿은 플레이스홀더
  - 엔드포인트: <메서드 경로 ...>
  - 호출예시: curl ...
  - 응답해석: <핵심 응답 필드 설명>
  ```

- [ ] **예시 질문** (괄호 = 발화 처리기 → 실측 결과)
  - **<카테고리> (<처리기> → <action>)**
    1. <입력 예시>

- [ ] **호출 예시**

  ```bash
  curl ...
  ```
  ```json
  { ...실제 응답(trim)... }
  ```
````

## 검증 체크리스트

````bash
f=docs/<name>.md
grep -c '^- \[ \]' "$f"     # 섹션(체크박스) 수
grep -c '```' "$f"          # 코드펜스(짝수=균형)
grep -c '^# ' "$f"          # H1 (0 이어야)
# + 모든 ```json 블록 파싱 OK, 시크릿 실값·미치환 플레이스홀더 0 확인
````

- [ ] 체크박스·코드펜스 균형, H1 0
- [ ] 모든 json 블록 파싱 OK
- [ ] 시크릿 실값·미치환 플레이스홀더(`<...>`, `__X__`) 잔존 0
- [ ] 예시 action 라벨이 라이브 실측과 일치
- [ ] GitHub 미리보기에서 토글·코드 정상 렌더
